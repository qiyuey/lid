# Lidless — contributor & agent notes

## Always verify locally before opening a PR

GitHub Actions CI is currently unavailable (account billing), so **local
verification is the gate**. Run the same checks CI would and make sure they're
green *before* creating any PR.

```bash
# Regenerate the Xcode project from project.yml (the source of truth —
# Lidless.xcodeproj is gitignored) and run the unit tests.
xcodegen generate
xcodebuild test -scheme Lidless-CI -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO
```

`| xcbeautify` is optional (prettier output); the run must end in
`** TEST SUCCEEDED **`.

For app/UI changes, also confirm the app builds:

```bash
xcodebuild build -scheme Lidless -destination 'generic/platform=macOS' \
  -configuration Debug CODE_SIGNING_ALLOWED=NO
```

Only open the PR once these pass locally. When CI is restored (or the repo is
public, where Actions runners are free), this stays the pre-PR smoke test — CI
just runs the same `Lidless-CI` scheme.

## Project basics

- **`project.yml` is the source of truth.** `Lidless.xcodeproj` is generated by
  `xcodegen generate` and gitignored — never hand-edit the project; edit
  `project.yml` and regenerate.
- **Debug builds use bundle id `com.nghialuong.lidless.dev`** so a dev build
  coexists with the released app (separate UserDefaults, login item, and
  privileged helper). Release uses `com.nghialuong.lidless`.
- **Release pipeline:** `scripts/release.sh` (archive → notarize → DMG →
  EdDSA-signed Sparkle appcast). Run on a Mac with the Developer ID cert and the
  `lidless-notary` keychain profile.
